▶ 30 秒上手›
  1. 在左侧「武将属性」填 id、name、hp、atk、def、移动/攻击范围。
  2. 在右侧「技能列表」点「+ 添加」，写 filter 与 content 代码。
  3. 点「✓ 提交」上传到服务器。
  4. 回到游戏首页，DIY 武将会出现在选将池中。

▶ 被动触发时机一览›
  • turnStart — 回合开始时触发（己方回合，可加护盾/buff）
  • turnEnd — 回合结束时触发（己方回合）
  • onKill — 击杀敌人时触发（context.target 为被击杀者）
  • onKilled — 自身被击杀时触发（context.killer 为击杀者）
  • onAttack — 发起攻击时触发（context.target 为攻击目标）
  • onAttacked — 被攻击后触发（context.attacker 为攻击者，context.damage 为受到的伤害）
  • onMove — 移动结束后触发（context.from/context.to 为起止坐标）
  • onHeal — 被治疗时触发（context.healer 为治疗者，context.amount 为治疗量）
  • onSkillCast — 任意技能发动后触发（context.caster 为施法者，context.skill 为技能）
▶ 限定技与临时技能›

限定技 limited：在技能定义中设置 limited: true 即可将技能标记为限定技。限定技每局游戏只能使用一次，使用后按钮变为禁用状态并显示"已用"。

// DIY 技能定义示例
{
  id: "diy_ganning_jieying",
  name: "劫营",
  type: "主动",
  limited: true,    // 限定技：本局仅一次
  cooldown: 0,
  desc: "限定技。突袭至目标位置..."
}

临时技能 Effect.gainTmpSkill：给棋子临时添加一个技能，支持多种过期方式。

// 持续到下回合开始（回合开始时递减）
Effect.gainTmpSkill(actor, 'skillId', { turnsAtStart: 1 });

// 持续2回合（每回合结束时递减）
Effect.gainTmpSkill(actor, 'skillId', { turns: 2 });

// 使用技能后消失（触发 onSkillCast 事件时移除）
Effect.gainTmpSkill(actor, 'skillId', { expiresOn: 'onSkillCast' });

// 第5回合结束时消失
Effect.gainTmpSkill(actor, 'skillId', { expiresAtTurn: 5 });

// 手动移除
Effect.loseTmpSkill(actor, tmpId);
Effect.loseAllTmpSkills(actor);

▶ AI 提示 aiHint 详解›

aiHint 是技能的可选字段，告诉 AI 如何评估和使用此技能。

💡 自动推断模式（推荐）：留空 type 字段时，系统会通过 SkillAnalyzer 自动分析技能 content 源码，推断类型/目标/威力等语义。新技能无需手动填写即可被 AI 正确使用。只有当自动推断不准确时，才需要手动覆盖。

字段一览（所有字段均可选，留空走自动推断）：

  • type（手动覆盖时必填）— 技能类型：damage伤害 / heal治疗 / buff增益 / debuff减益 / control控制 / teleport位移 / summon召唤 / mixed混合
  • target（手动覆盖时必填）— 目标类型：enemy敌方单体 / ally友方单体 / cell格子 / self自身 / none无目标 / aoe_enemy范围敌方 / aoe_ally范围友方
  • power（0-200，默认30）— 威力值，影响基础评分。伤害类对应伤害量，治疗类对应治疗量
  • priority（1-10，默认5）— 优先级，越高越优先使用
  • condition（默认 always）— 使用条件，不满足时 AI 不会用此技能：
    • always 总是评估
    • enemy_near 附近 4 格内有敌人
    • enemy_in_range 技能范围内有敌人
    • ally_injured 有友军血量低于阈值
    • self_low_hp 自身血量低于阈值
    • self_full_hp 自身满血
    • has_target 有合法目标
  • preferTarget（默认空）— 目标偏好策略：
    • low_hp 优先低血目标（补刀）
    • high_threat 优先高威胁目标
    • injured_ally 优先残血友军（治疗）
    • nearest 优先最近敌人
    • caster 优先法师型单位（低防高攻）
  • minTargets（0-10，默认0）— 范围技至少命中 N 个目标才使用
  • avoidSelf（bool，默认 false）— 是否避免选自己作为目标
  • hpThreshold（0-100，默认0）— 血量阈值百分比，配合 self_low_hp/ally_injured 使用，0 表示用默认值（40%/70%）
  • notes（字符串）— 设计备注，不参与 AI 计算，仅作记录

评分公式：score = power * (0.5 + priority / 10)，再根据 type/condition/minTargets 调整。

示例：关羽武圣（额外攻击+回血）：

aiHint: {
  type: 'damage',
  target: 'self',        // 自身增益
  power: 50,
  priority: 5,
  condition: 'enemy_near', // 附近有敌时才用
  preferTarget: 'high_threat'
}

提示：aiHint 只影响 AI 决策，不影响技能实际效果。技能效果完全由 content 代码决定。

▶ 选项模态框 chooseOption›

Effect.chooseOption(actor, opts) 显示一个模态框让玩家选择，AI 模式下自动选择最优项。适合「分支选择」类技能（如：强攻 / 防御 / 治疗三选一）。

参数结构：

const choice = await Effect.chooseOption(actor, {
  title:    '请选择效果',        // 模态框标题
  hintText: '发动分支技能！',    // 战报提示（可选）
  options: [                    // 选项数组
    {
      label:  '强攻',           // 显示文本（必填）
      desc:   '+30 攻击力',     // 描述（可选）
      value:  'atk',            // 选项值（可选）
      aiScore: 60,              // AI 评分（默认 0，越高越优）
      aiCondition: function(actor) {  // AI 可用条件（可选）
        return true;
      }
    },
    {
      label:  '治疗',
      desc:   '回复 50 生命',
      value:  'heal',
      aiScore: 80,
      aiCondition: function(actor) {
        return actor.hp < actor.maxHp * 0.5;  // 仅低血时 AI 会选
      }
    }
  ],
  aiPicker: null  // 自定义 AI 选择函数 (actor, options) => option（可选）
});

if (choice) {
  // choice = { label, desc, value, aiScore, _index }
  if (choice.value === 'atk') {
    Effect.modifyAttack(actor, 30, 2);
  } else if (choice.value === 'heal') {
    Effect.heal(actor, 50);
  }
  global.Game.log(actor.name + ' 选择了 ' + choice.label);
}

AI 选择逻辑（自动执行）：

  1. 若提供 aiPicker 函数，优先调用它
  2. 否则过滤掉 aiCondition 返回 false 的选项
  3. 按 aiScore 排序，最高分当选（同分随机）
  4. 全部不可用则返回 null（AI 放弃此技能）

返回值：选中选项对象（含 label/desc/value/aiScore 等 + _index）或 null（玩家取消 / AI 无可用项）。

提示：必须用 await 等待玩家选择。AI 模式下会立即返回不阻塞。

▶ 多步技能与 AI 支持›

多步技能指 content 代码中包含一个或多个 await Effect.chooseXxx() 的技能，例如「先选格子 → 再对范围内敌人造成伤害」或「让玩家从 3 个效果中选 1 个」。

AI 自动支持：AI 执行技能时，会设置 Effect._aiContext 并贯穿整个 content 的 async 执行。content 内部的 chooseCell/chooseEnemy/chooseAlly/chooseOption 检测到 AI 模式后会立即返回 AI 选择的结果（不弹窗、不阻塞），让多步流程顺畅跑完。

AI 选择依据：

  • chooseCell/chooseEnemy/chooseAlly：根据 aiHint.target / preferTarget / avoidSelf 自动选最优格子或棋子
  • chooseOption：根据 aiScore / aiCondition / aiPicker 自动选最优选项

content 返回 false 的约定：

如果 AI（或玩家）在中间步骤取消选择（chooseXxx 返回 null），content 应该 return false。AI 引擎检测到 result === false 时不会应用冷却，并继续评估其他行动。

// 多步技能示例：先选格子，再对范围内敌人造成伤害
async function content(actor) {
  const cell = await Effect.chooseCell(actor, {
    range: { shape: '+', n: 3 },
    mustEmpty: false,    // 可以选到敌人
    hintText: '请选择目标格'
  });
  if (!cell) return false;  // 取消则不消耗技能

  const cells = Range.cellsInRange('r', 1, cell.x, cell.y, { includeSelf: true });
  let hit = 0;
  for (const c of cells) {
    const t = Game.pieceAt(c.x, c.y);
    if (t && t.alive && t.side !== actor.side) {
      Effect.damage(actor, t, 50);
      hit++;
    }
  }
  if (hit === 0) return false;  // 没命中任何敌人也算未使用

  actor.skilled = true;
  return true;
}

自定义 AI 逻辑：在 content 内可调用 Effect.aiContext() 检测当前是否 AI 模式，并写分支逻辑：

async function content(actor) {
  let target;
  const aiCtx = Effect.aiContext();
  if (aiCtx) {
    // AI 模式：自定义选目标逻辑
    target = Game.pieces.filter(p => p.alive && p.side !== actor.side)
      .sort((a, b) => a.hp - b.hp)[0];  // 选血量最低的敌人
  } else {
    // 玩家模式：弹窗选择
    target = await Effect.chooseEnemy(actor, { range: { shape: '+', n: 5 } });
  }
  if (!target) return false;
  Effect.damage(actor, target, 80);
  actor.skilled = true;
}

提示：尽量用 chooseXxx 让 AI 自动决策；只有需要复杂自定义逻辑时才用 aiContext() 分支。

▶ ★ 新效果 API 详解›

以下为新增的高性价比 Effect，每个附带完整示例：

1. 不屈 Effect.undying(target, stacks)

致命伤害时保留 1 点生命，消耗一层标记。stacks 可叠加（挡多次致命伤）。

// 主动技能：给己方核心武将2层不屈
const ally = await Effect.chooseAlly(actor, {
  range: { shape: 'r', n: 2 },
  hintText: '请选择友方'
});
if (!ally) return false;
actor.skilled = true;
Effect.undying(ally, 2);
return true;

2. 复活 Effect.revive(target, turns, ratio)

死亡后延迟 turns 回合，以 ratio 比例血量复活。位置被占时自动找最近空位。

// 被动技能（turnStart）：每回合开始给自己复活标记
// trigger: turnStart
Effect.revive(actor, 3, 0.3); // 死后3回合以30%血量复活
global.Game.log(actor.name + ' 祭起复活术！');

3. 隐身 Effect.stealth(target, turns)

隐身期间无法被选为攻击/技能目标（AI 和玩家都无法选中）。棋子半透明显示。

// 主动技能：自身隐身2回合
actor.skilled = true;
Effect.stealth(actor, 2);
return true;

配合 Effect.isUntargetable(piece) 可在代码中检查某棋子是否隐身：

if (Effect.isUntargetable(target)) {
  global.Game.log(target.name + ' 处于隐身，无法选中！');
  return false;
}

4. 标记引爆 Effect.detonate(target, markName, callback)

消耗目标身上的指定标记，执行回调函数 callback(piece, markData)，然后移除标记。让"叠加标记→引爆"的 Combo 流成为可能。

// 主动技能：先叠加3层"破绽"标记，再引爆
const target = await Effect.chooseEnemy(actor, {
  range: { shape: '+', n: 3 }
});
if (!target) return false;
actor.skilled = true;

// 检查是否有"破绽"标记
var data = Effect.getMarkData(target, 'flaw');
if (data && data.stacks >= 3) {
  // 引爆：每层造成20点真实伤害
  Effect.detonate(target, 'flaw', function(piece, d) {
    Effect.damage(actor, piece, d.stacks * 20, { ignoreDef: true });
  });
  global.Game.log('【破绽引爆】造成 ' + (data.stacks * 20) + ' 真实伤害！');
} else {
  // 叠加标记
  var cur = data ? data.stacks : 0;
  Effect.unmark(target, 'flaw');
  Effect.mark(target, 'flaw', {
    display: '破',
    modifiers: { flawStacks: cur + 1 },
    data: { stacks: cur + 1 }
  });
  global.Game.log(target.name + ' 身上破绽+1（当前 ' + (cur + 1) + ' 层）');
}
return true;

5. 属性偷取 Effect.stealStat(actor, target, stat, amount)

永久从 target 偷取 amount 点属性给 actor。stat 为 'atk' 或 'def'。不会恢复！

// 被动技能（onAttack）：攻击时偷取5点攻击力
// trigger: onAttack
if (!context || !context.target) return;
var t = context.target;
if (t.alive) {
  Effect.stealStat(actor, t, 'atk', 5);
}

6. 陷阱 Effect.placeTrap(x, y, opts)

在指定格子放置陷阱，敌方棋子踩到时自动触发。opts.type 支持：damage(伤害) / stun(眩晕) / teleport(随机传送) / poison(中毒)。

// 主动技能：在目标格布雷
const cell = await Effect.chooseCell(actor, {
  range: { shape: '+', n: 3 },
  mustEmpty: true,
  hintText: '请选择布雷位置'
});
if (!cell) return false;
actor.skilled = true;
Effect.placeTrap(cell.x, cell.y, {
  type: 'damage',   // 类型：damage/stun/teleport/poison
  damage: 40,       // 伤害量（damage/poison类型用）
  turns: 3,         // 3回合后自动消失
  owner: actor,     // 陷阱归属
  side: 'enemy'     // 敌方触发
});
return true;

清除陷阱：Effect.clearTraps({owner: actor}) 或 Effect.clearTraps({x, y}) 或 Effect.clearTraps()（清全部）。

7. 伤害分摊 Effect.linkDamage(allyA, allyB, turns, ratio)

链接两个友军，任一方受伤时转移 ratio 比例伤害给另一方。分摊伤害会正确触发不屈/护盾/死亡事件。

// 主动技能：链接自己和一名友军，分摊50%伤害
const ally = await Effect.chooseAlly(actor, {
  range: { shape: 'r', n: 3 },
  hintText: '请选择链接友军'
});
if (!ally) return false;
actor.skilled = true;
Effect.linkDamage(actor, ally, 3, 0.5); // 3回合内分摊50%
return true;
